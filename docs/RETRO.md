## 可以更好的

| 项目 | 改进方向 |
|------|---------|
| **前端架构** | index.html 已近 700 行，应考虑拆分为模块或引入构建工具 |
| **内容管理** | 景点/笔记数据硬编码在 HTML 里，Markdown→JS 脚本虽可用但未投入实战 |
| **图片质量** | Pexels 自动下载的图片需人工筛选，部分不够精准（如铜梁火龙搜出普通舞龙） |
| **测试覆盖** | 仅后端 API 测试，前端完全无测试 |
| **无 HTTPS** | Let's Encrypt 免费 SSL 始终未配置 |

## 如果再选一次

1. **先用纯 HTML 验证，不要急着上 Next.js** — 我们走了 Next.js→放弃→纯HTML 的弯路，白白浪费了 SWC 兼容性调试时间
2. **图片应该一开始就规划好** — 先有了 46 张的清单再去下载，比边做边找高效得多
3. **.env 文件必须检查 Python 版本兼容性再部署** — 服务器 Python 3.8 不支持 `list[str]` 语法，浪费了 30 分钟
