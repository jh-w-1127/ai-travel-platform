# 产品需求文档 (PRD)

## 项目背景

中国入境游市场快速增长，但海外游客面临语言障碍、信息碎片化、支付困难三大核心痛点。现有解决方案（携程英文版、TripAdvisor）缺乏对中国本地深度内容的覆盖，无法提供个性化行程规划。

## 目标用户

| 用户画像 | 特征 |
|---------|------|
| 独立旅行者 | 30-50 岁，欧美/东南亚，英语为主，预算中等 |
| 家庭游客 | 带老人/小孩，需要安全+便利+中文辅助 |
| 商务+休闲 | 展会/出差之余抽 2-3 天游玩，时间紧 |

## 核心痛点

1. **信息不对称**：Google 搜重庆攻略要么是 10 年前的游记，要么是机翻中文
2. **支付障碍**：不知道 Alipay 能绑 Visa，不知道哪里收现金
3. **行程焦虑**：景点之间怎么走？一天怎么看最值？没人告诉我
4. **内容枯萎**：景点介绍像说明书，没有故事，没有"为什么值得去"

## 产品目标

| 目标 | 衡量标准 |
|------|---------|
| 让外国游客能在 5 分钟内看到一个完整的重庆旅行方案 | 从打开网站到看到行程 < 3 次点击 |
| AI 行程规划覆盖重庆核心景点 | 13 个景点 + 12 篇笔记 |
| 解决支付+交通+签证最常见的问题 | AI客服可回答 10+ 类旅行问题 |
| 内容质量达到"值得收藏"水平 | 每篇 200-300 字，含故事+实用信息 |

## 功能范围

### V1 (MVP) — 已完成

| 功能 | 说明 |
|------|------|
| 景点浏览 | 13 景点卡片 + 详情弹窗 + 实景图片 |
| 旅行笔记 | 12 篇深度内容 + 景点关联 |
| AI 行程计划 | 模板引擎 + DeepSeek Function Calling |
| AI 旅行助手 | 关键词匹配 + LLM 双模式 |
| 转人工客服 | 支持表单提交 + admin 后台 |
| 费用预估 | 酒店/餐饮/交通/景点分项报价 |

### V2 — 待规划

- 英文版全站翻译
- Stripe 支付接入
- AI 智能编排行程（LLM 替换模板引擎）
- RAG 知识库升级 AI 客服
- 多城市扩展（成都、西安、昆明）
